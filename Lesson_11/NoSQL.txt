1. В базе данных Redis подберите коллекцию для подсчета посещений с определенных IP-адресов.

Простейшее предложение - в качестве ключа использовать строку с IP-адресом (напр., '78.30.254.48'),
значение присваивать командой INCR. При первом подключении будет присвоено значение 1, затем оно будет 
увеличиваться на единицу с каждым подключением. На каждый уникальный IP-адрес будет создаваться 
отдельный ключ (в теории, количества ключей должно хватить на не супер нагруженную систему).
Итоговая команда будет выглядеть так:
INCR '78.30.254.48'
Узнать количество подключений можно командой
GET '78.30.254.48'
Конечно, я не думаю, что кто-то воспользуется этим, т.к. обычно задачи стоят более масштабные. Но для 
конкретной задачи из ДЗ этого достаточно, как мне кажется.



2. При помощи базы данных Redis решите задачу поиска имени пользователя по электронному адресу и наоборот, 
поиск электронного адреса пользователя по его имени.

Предложение опять простейшее - для каждой пары имя:имейл создаем две записи. Одна запись с ключем имя и 
значением имейл, вторая с ключем имейл и значением имя. Командой GET имя получим имейл, командой GET имейл
получим имя =)
